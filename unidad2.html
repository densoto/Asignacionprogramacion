Acceso a datos y ADO.NET

Para el acceso a datos en Visual Basic 2017 se utiliza el modelo de acceso de datos que proporciona la plataforma .NET Framework: ADO.NET

ADO.NET es un conjunto de interfaces, clases, estructuras y enumeraciones que permiten el acceso a datos desde la plataforma.NET.

ADO.NET permite un modo de acceso desconectado a los datos, estos pueden provenir de multiples fuentes de datos con diferentes arquitecturas de almacenamiento.

Los proveedores de datos .NET Framework se utilizan para conectarse a una base de datos, ejecutar comandos y recuperar resultados. Estos resultados son procesados directamente y se colocan en un DataSet con el fin que el usuario pueda verlos cuando los necesite, pueden combinarse con datos de varios origenes o utilizarse de forma remota entre niveles.

.NET Framework Proveedor de datos para SQL Server

Proporciona acceso a datos para Microsoft SQL Server. Utiliza el espacio de nombres System.Data.SqlClient.

Proveedor de datos.NET Framework para OLE DB

Para origenes de datos que se exponen mediante OLE DB. Utiliza el espacio de nombres System.Data.Odbc.

.NET Framework para Oracle

Para origenes de datos Oracle. El proveedor de datos .NET Framework para Oracle es compatible con la version 8.1.7 y posteriores del software de cliente de Oracle y utiliza el espacio de nombres System.Data.OracleClient.

Validar Datos

La validacion se refiere al proceso de confirmacion de los datos introducidos por parte del usuario. La validacion tiene como objetivo prevenir que se ingresen datos incorrectos o que no se introduzcan todos los datos necesarios.

En nuestro ejemplo, es posible que necesitemos asegurarnos que el usuario introduzca obligatoriamente el codigo y nombre del cliente; de igual forma, necesitariamos asegurarnos que el codigo solamente acepte numeros y que el TextBox que recoge el nombre solo acepte letras.
